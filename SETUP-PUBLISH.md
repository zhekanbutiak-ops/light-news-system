# Покрокова налаштування публікацій у Telegram

## Що вже є в проєкті

- **Новини з сайту** → публікуються в твій канал **кожні 15 хв** (`/api/auto-publish`).
- **Пости про пуски/тривоги** з TG-каналів → публікуються **тільки коли джерело щось опублікувало** (`/api/auto-publish-threats`, перевірка кожні 2 хв). Підпис каналу типу «ПІДПИСАТИСЯ | ППО UA РАДАР» автоматично прибирається.

---

## Крок 1. Змінні оточення (Vercel)

У проєкті на Vercel → **Settings** → **Environment Variables** додай (або перевір, що вже є):

| Змінна | Опис |
|--------|------|
| `CRON_SECRET` | Секретний рядок (наприклад згенеруй на random.org). Його Vercel підставляє в заголовок при виклику cron. |
| `TELEGRAM_BOT_TOKEN` | Токен бота від [@BotFather](https://t.me/BotFather). |
| `TELEGRAM_CHAT_ID` | ID твого каналу (наприклад `-1001234567890`). Як отримати: перешли повідомлення з каналу боту [@userinfobot](https://t.me/userinfobot) або додай бота в канал як адміна і подивись ID. |
| `GROQ_API_KEY` | Ключ з [console.groq.com](https://console.groq.com) — потрібен для опису новин у `/api/auto-publish`. |

Після змін змінних зроби **Redeploy** проєкту.

---

## Крок 2. Отримати RSS для каналу про пуски

1. Відкрий один із сервісів:
   - **https://ch2rss.fflow.net/**
   - або **https://tg-channel-to-rss.vercel.app/**
2. Вкажи **публічний** Telegram-канал (наприклад `ppo_ua_radar` або повне посилання).
3. Сервіс видасть **RSS-посилання**. Скопіюй його (виглядає приблизно так: `https://ch2rss.fflow.net/...` або подібне).

---

## Крок 3. Підставити RSS у код

1. Відкрий у проєкті файл **`app/api/auto-publish-threats/route.ts`**.
2. Знайди масив **`THREAT_SOURCES`** (приблизно рядки 21–24).
3. Розкоментуй рядок-приклад і встав **своє RSS-посилання** та назву блоку:

```ts
const THREAT_SOURCES = [
  { name: "⚠️ ПУСКИ", url: "ТВОЄ_RSS_ПОСИЛАННЯ" },
  // Можна додати ще канали:
  // { name: "⚠️ ТРИВОГИ", url: "https://..." },
];
```

4. Збережи файл, закоміть і запуш на GitHub. Після деплою на Vercel cron почне викликати `/api/auto-publish-threats` кожні 2 хвилини.

---

## Крок 4. Перевірка cron на Vercel

- У Vercel: **Settings** → **Crons** — мають бути два завдання:
  - `*/15 * * * *` → `/api/auto-publish` (новини з сайту кожні 15 хв).
  - `*/2 * * * *` → `/api/auto-publish-threats` (перевірка каналів про пуски кожні 2 хв).

Vercel сам викликає ці URL з заголовком `Authorization: Bearer <CRON_SECRET>`.

---

## Крок 5. Ручна перевірка (опційно)

Щоб перевірити без очікування cron:

```bash
# Новини (одна публікація з TSN)
curl -H "Authorization: Bearer ТВІЙ_CRON_SECRET" "https://ТВІЙ_ДОМЕН.vercel.app/api/auto-publish"

# Загрози (якщо є новий пост у THREAT_SOURCES за останні ~2.5 хв)
curl -H "Authorization: Bearer ТВІЙ_CRON_SECRET" "https://ТВІЙ_ДОМЕН.vercel.app/api/auto-publish-threats"
```

У відповіді побачиш `success` / `ok` та деталі.

---

## Підсумок

| Що | Коли | Де налаштовується |
|----|------|--------------------|
| Новини з сайту (TSN тощо) | Кожні 15 хв | Працює автоматично, потрібні лише змінні оточення |
| Пости про пуски з TG-каналу | Тільки коли канал щось опублікував (перевірка кожні 2 хв) | Додати RSS у `THREAT_SOURCES` у `app/api/auto-publish-threats/route.ts` |

Підпис каналу («ПІДПИСАТИСЯ | ППО UA РАДАР») з текстів про пуски вже прибирається в коді автоматично.
