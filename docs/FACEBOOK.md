# Підключення сторінки Facebook

Щоб пости з сайту (ранковий дайджест, хвилина мовчання, автопублікація новин, свята) з’являлись на вашій Facebook-сторінці, потрібно додати **дві змінні середовища**.

---

## 1. Що потрібно

| Змінна | Опис |
|--------|------|
| `FACEBOOK_PAGE_ID` | ID вашої сторінки Facebook |
| `FACEBOOK_PAGE_ACCESS_TOKEN` | Довготривалий токен доступу до сторінки (щоб публікувати пости) |

---

## 2. Де вставити

### Локально (для тестів)
У корені проєкту створіть або допишіть у файл **`.env.local`**:

```
FACEBOOK_PAGE_ID=ваш_page_id
FACEBOOK_PAGE_ACCESS_TOKEN=ваш_токен
```

### На Vercel (production)
1. Відкрийте проєкт на [vercel.com](https://vercel.com) → **Settings** → **Environment Variables**.
2. Додайте дві змінні:
   - **Name:** `FACEBOOK_PAGE_ID` → **Value:** ваш ID сторінки.
   - **Name:** `FACEBOOK_PAGE_ACCESS_TOKEN` → **Value:** ваш Page Access Token.
3. Оберіть середовище (Production / Preview) і збережіть.
4. Зробіть **Redeploy**, щоб нові змінні підхопились.

---

## 3. Покроково: де взяти FACEBOOK_PAGE_ID (ID сторінки)

**Спосіб А — з самої сторінки Facebook**

1. Відкрийте **facebook.com** і увійдіть у свій акаунт.
2. Перейдіть на **вашу сторінку** (не профіль, саме сторінка, яку ви створили).
3. Натисніть зліва **«Про сторінку»** (або в меню три крапки → **About** / **Page transparency**).
4. Прокрутіть вниз до блоку **«ID сторінки»** (Page ID).
5. Скопіюйте число — це і є **FACEBOOK_PAGE_ID** (наприклад: `1085123456789012`).

**Спосіб Б — через Meta for Developers**

1. Відкрийте в браузері: **[developers.facebook.com/tools/explorer](https://developers.facebook.com/tools/explorer)**.
2. У верхньому лівому куті оберіть **свій додаток** (або створіть новий — див. розділ про токен нижче).
3. Під полем з посиланням є випадаючий список **«User or Page»**.
4. Натисніть на нього і виберіть **вашу сторінку** зі списку.
5. У полі **«Access Token»** з’явиться токен — поки що його не копіюйте.
6. У самому запиті (GET) замість `me` можна подивитись ID: змініть шлях на `me?fields=id` і натисніть **Submit** — у відповіді буде `"id": "1085123456789012"`. Це і є **FACEBOOK_PAGE_ID**.

## 4. Покроково: де взяти FACEBOOK_PAGE_ACCESS_TOKEN (токен сторінки)

**Крок 1. Створити додаток Meta (якщо ще немає)**

1. Відкрийте: **[developers.facebook.com/apps](https://developers.facebook.com/apps)**.
2. Натисніть **«Створити додаток»** (Create App).
3. Оберіть тип **«Бізнес»** (Business) або **«Інше»** (Other) → **Далі**.
4. Введіть назву додатка (наприклад: Light News) і контактний email → **Створити додаток**.
5. Додаток з’явиться у списку — натисніть на нього.

**Крок 2. Відкрити Graph API Explorer**

1. Відкрийте: **[developers.facebook.com/tools/explorer](https://developers.facebook.com/tools/explorer)**.
2. Зверху зліва у полі **«Meta App»** оберіть **ваш додаток** (той, що щойно створили).

**Крок 3. Додати дозволи (permissions)**

1. Під полем з URL запиту знайдіть **«Permissions»** або **«Add a Permission»**.
2. Натисніть і додайте ці права:
   - **pages_manage_posts** — публікація постів;
   - **pages_read_engagement** — читання взаємодій;
   - **pages_show_list** — список сторінок.
3. Збережіть / застосуйте дозволи.

**Крок 4. Отримати User Access Token**

1. Натисніть кнопку **«Generate Access Token»** (або **Get User Access Token**).
2. У вікні увійдіть у Facebook, якщо попросить.
3. Поставте галочки для доступу до **вашої сторінки** (наприклад: «Light News»).
4. Натисніть **«Continue»** / **«Продовжити»** → **«OK»**.
5. У полі **«Access Token»** з’явиться довгий рядок — це **User Access Token** (короткотривалий, ~1 година).

**Крок 5. Отримати Page Access Token (токен саме сторінки)**

1. У тому ж Graph API Explorer під полем запиту знайдіть випадаючий список **«User or Page»**.
2. Натисніть на нього.
3. Зі списку оберіть **вашу сторінку** (не свій профіль).
4. Після вибору сторінки **токен у полі «Access Token» автоматично зміниться** — це вже **Page Access Token**.
5. Натисніть **«Copy»** біля токена або виділіть і скопіюйте весь рядок.
6. Цей рядок вставте в змінну **FACEBOOK_PAGE_ACCESS_TOKEN** у `.env.local` або в Vercel.

**Важливо:** цей токен може жити недовго (1–2 години). Якщо пізніше пости перестануть публікуватись — потрібно отримати **довготривалий** токен (крок 6).

**Крок 6 (опційно). Довготривалий Page Access Token (~60 днів)**

1. У своєму додатку на [developers.facebook.com/apps](https://developers.facebook.com/apps) відкрийте **Settings** → **Basic** — скопіюйте **App ID** та **App Secret**.
2. У браузері відкрийте посилання (підставте свої значення замість ВАШ_APP_ID, ВАШ_APP_SECRET та USER_ACCESS_TOKEN з кроку 4):
   ```
   https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=ВАШ_APP_ID&client_secret=ВАШ_APP_SECRET&fb_exchange_token=USER_ACCESS_TOKEN
   ```
3. У відповіді скопіюйте значення **access_token** — це довготривалий **User** токен.
4. Відкрийте друге посилання (підставте скопійований токен замість ДОВГОТРИВАЛИЙ_USER_TOKEN):
   ```
   https://graph.facebook.com/v21.0/me/accounts?access_token=ДОВГОТРИВАЛИЙ_USER_TOKEN
   ```
5. У відповіді знайдіть об’єкт з вашою сторінкою (за назвою або `id` сторінки). У ньому поле **access_token** — це і є **довготривалий Page Access Token**. Його вставте в **FACEBOOK_PAGE_ACCESS_TOKEN**.

---

## 5. Перевірка

Після збереження змінних:

- **Автопублікація** (кожні 30 хв у день) — той самий пост, що й у Telegram, дублюється на Facebook.
- **Ранковий дайджест** (07:00 Київ) — текст дайджесту публікується і на сторінку.
- **Хвилина мовчання** (07:59/08:59 Київ) — пост про хвилину мовчання.
- **Свята** (daily-holiday) — пост про свято.

Якщо змінних немає або токен невалідний, пости в Facebook не публікуються (помилки лише в логах сервера), Telegram і сайт працюють як раніше.
